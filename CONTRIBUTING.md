# Contributing

Thank you for your interest in contributing to QR-Code-Generator! We welcome improvements, bug fixes, tests and documentation updates.

Guidelines

- Fork the repository and create a feature branch using a descriptive name: `feature/your-feature` or `fix/issue-number`.
- Keep changes small and focused. Add tests for new features or bug fixes.
- Follow PEP8 and maintain the existing code style.
- Ensure all tests pass locally (`pytest`).

Pull request process

1. Open a PR against `main` describing the change and motivation.
2. Reference any related issue in the PR description.
3. Ensure CI passes (GitHub Actions runs tests automatically).

Code of conduct

Be respectful and constructive. Report any policy violations by opening an issue.
